# User Flows & Functionality

## Primary User Types
1. **Prospective Students**: Seeking admission information
2. **Current Students**: Accessing portal and resources
3. **Staff Members**: Administrative and teaching functions
4. **Parents/Guardians**: Information gathering and enrollment

## Key User Journeys

### Prospective Student Journey
1. **Discovery**: Homepage → About → Academics
2. **Interest**: Admissions page → Requirements review
3. **Application**: Student Portal → Account creation → Enrollment form
4. **Follow-up**: Portal login → Application status check

### Current Student Journey
1. **Access**: Student Portal login
2. **Dashboard**: View personal information and updates
3. **Resources**: Access academic materials and schedules
4. **Communication**: Contact forms and support

### Staff Journey
1. **Authentication**: Staff Portal login
2. **Dashboard**: LeaderHome.html interface
3. **Management**: Student records and administrative tasks
4. **Communication**: Internal messaging and updates

## Portal Systems

### Student Portal Features
- Account creation and authentication
- Enrollment application forms
- Application status tracking
- Personal dashboard
- Resource access

### Staff Portal Features
- Administrative dashboard (LeaderHome.html)
- Student management
- Academic oversight
- Communication tools

## Navigation Structure
- **Primary Navigation**: Home, About, Academics, Admissions, Events, Student Life, Student Portal, Staff Portal
- **Responsive Design**: Hamburger menu for mobile devices
- **Cross-linking**: Strategic internal links between related sections
- **Call-to-Actions**: Strategic placement of enrollment and contact buttons

## Form Systems
- **Student Registration**: Multi-step process with document upload
- **Staff Authentication**: Secure login system
- **Contact Forms**: Multiple contact points throughout site
- **Enrollment Forms**: Separate forms for O-Level and A-Level
